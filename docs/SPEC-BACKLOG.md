## P1 — доверие и конверсия (текущий фокус)

### P1.1 FAQ + статус занятости на главной

**Приоритет:** 1  
**Оценка:** ~1–2 ч кода + тексты от владельца

#### Зачем

Снимает типовые вопросы и страхи до перехода на `/contact`. Повышает конверсию визитки фрилансера.

#### Статус занятости

**Сейчас:** в `src/shared/consts/contact.ts` поле `status: 'Принимаю небольшие заказы'` — слишком общее.

**Сделать:** расширить константы (пример):

```ts
// src/shared/consts/contact.ts
export const SITE_CONTACT = {
  // ...existing
  availability: 'Свободен',           // Свободен | 1 слот | Занят до …
  availabilityDetail: 'Могу начать с …', // опционально
  responseTime: 'Отвечаю за 1–2 рабочих дня',
} as const
```

**UI:**

- Показать в **hero**, в строке eyebrow: `{city} · {availability} · {responseTime}`
- Опционально цветной бейдж: зелёный / жёлтый / серый по статусу

**Файлы:**

- `src/shared/consts/contact.ts`
- `src/pages/AboutMe/ui/AboutMe.tsx`
- `src/pages/AboutMe/ui/styles.module.scss`

#### FAQ

**Формат:** аккордеон Bootstrap (`Accordion`) — компактно на мобилке.

**Данные:** вынести в `src/shared/consts/faq.ts`:

```ts
export interface FaqItem {
  id: string
  question: string
  answer: string
}

export const FAQ_ITEMS: FaqItem[] = [ ... ]
```

**Рекомендуемые вопросы (черновики — правит владелец):**


| id          | Вопрос                              | Черновик ответа                                                                                             |
| ----------- | ----------------------------------- | ----------------------------------------------------------------------------------------------------------- |
| `timeline`  | Сколько времени займёт проект?      | Зависит от объёма. Лендинг — 1–2 недели, доработка — от нескольких дней. После обсуждения — оценка и сроки. |
| `location`  | Работаете только в Красноярске?     | База — Красноярск, проекты веду удалённо по России.                                                         |
| `price`     | Как формируется цена?               | Ориентиры на главной. Точная сумма — после ТЗ или короткого созвона.                                        |
| `prepay`    | Нужна ли предоплата?                | *Заполнить владельцем: % и условия*                                                                         |
| `process`   | Как начинаем работу?                | Заявка → уточнение задачи → оценка и сроки → согласование → разработка → сдача.                             |
| `revisions` | Сколько правок входит?              | *Заполнить: например 1–2 раунда по согласованному ТЗ*                                                       |
| `side-job`  | Это основная работа или подработка? | Совмещаю с основной работой, беру проекты с понятным объёмом и сроками.                                     |
| `handover`  | Передаёте исходники?                | Да, код и доступы — заказчику после оплаты.                                                                 |


**Порядок секций на главной (после внедрения):**

1. Hero (+ статус)
2. Услуги
3. Услуги и цены
4. **FAQ** ← новый
5. Для кого / Технологии / Формат работы *(часть можно сократить, если дублирует FAQ)*
6. CTA «Есть задача?»

**Критерии приёмки:**

- FAQ отображается на `/`, минимум 6 вопросов
- Аккордеон работает на мобилке
- Статус занятости виден в hero без прокрутки (desktop)
- Тексты редактируются только в `faq.ts` / `contact.ts`

## P2 — рост

### P2.1 Админка заявок

**Зачем:** смотреть `contact_requests` без SSH/psql.

**Scope MVP:**

- Маршрут `/admin` (не в меню, только по URL)
- Простой auth: пароль в `.env` (`ADMIN_PASSWORD`) → cookie/session
- Таблица: имя, контакт, сообщение, бюджет, дата, статус
- API: `GET /api/admin/requests`, `PATCH .../status`

**Файлы:** `server/src/routes/admin.ts`, `src/pages/Admin/`, миграция не нужна (таблица есть).

---

### P2.2 Блог

- Таблица `blog_posts` (миграция `002_blog_posts.sql`)
- Markdown или rich text в БД
- Маршрут `/blog`, `/blog/:slug`
- SEO: отдельные title/description на пост

---

### P2.3 Калькулятор стоимости

- Виджет на главной: тип проекта + опции → ориентир (не юридическое предложение)
- Данные из `pricing.ts` или отдельный конфиг
- CTA «Уточнить расчёт» → `/contact` с query-параметрами

---

### P2.4 Email-дайджест заявок

- Cron на VPS: `0 9 * * 1` — раз в неделю сводка на `NOTIFY_EMAIL`
- Скрипт `scripts/weekly-digest.sh` или endpoint + cron в docker

---

### P2.5 Pet-project для витрины

- Продукт с auth + БД (не CRUD на чужом API)
- Оформить как кейс в портфолио
- См. ROADMAP: учебные SPA (Posts list, Pokedex) **не показывать** заказчикам

---

### P2.6 Мультиязычность RU/EN

- i18n (react-i18next или аналог)
- Минимум: главная + контакты на EN для удалённых заказчиков

---

### P2.7 Lighthouse 90+

- Оптимизация аватара (сейчас ~4 MB в бандле — критично)
- Lazy load изображений, сжатие `myAvatar.jpeg`
- Проверка: PageSpeed Insights после правок

## Технические ориентиры

### Ключевые пути


| Назначение | Путь                                                     |
| ---------- | -------------------------------------------------------- |
| Главная    | `src/pages/AboutMe/ui/AboutMe.tsx`                       |
| Контакты   | `src/pages/Contact/`, `src/shared/consts/contact.ts`     |
| Портфолио  | `src/pages/Portfolio/`, `src/shared/consts/portfolio.ts` |
| Цены       | `src/shared/consts/pricing.ts`                           |
| Роутер     | `src/app/router/index.tsx`                               |
| Метрика    | `index.html`, `src/app/hooks/useYandexMetrika.ts`        |
| API        | `server/src/`                                            |
| nginx SSL  | `docker/nginx/conf.d/site.conf` (на сервере)             |
| Env prod   | `/opt/danilmakes/.env` (не в git)                        |


### Конвенции

- Контент — в `src/shared/consts/*.ts`, не хардкодить в JSX
- Стили — CSS modules рядом с компонентом
- После изменений: `npm run lint`, `npm run build`, `rsync-deploy`
- Коммиты: формат `feature-<описание>` + строки `-- Что сделано`

### Учебные демо (не для заказчиков)

Маршруты `/PostsList`, `/CodeExample/`* остаются в коде, но **не в меню**.  
В портфолио показывать: вёрстки, «Цены и ограничения», Divisions, «Отчёт по доходу», RacketMate, danilmakes.
