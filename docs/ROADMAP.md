# Roadmap: рекламная страница

## P0 — привлечение клиентов

- [x] Главная как лендинг: hero, CTA, блок услуг
- [x] Страница портфолио — карточки с описанием и ссылками на демо
- [ ] **2–3 сильных кейса** для витрины (см. чеклист ниже)
- [ ] Блок «Услуги и цены» (ориентиры)
- [ ] SEO: Open Graph, `robots.txt`, `sitemap.xml`
- [ ] Яндекс.Метрика
- [ ] Мобильная вёрстка (проверка offcanvas, формы, hero)

## P1 — доверие и конверсия

- [ ] Кейсы: задача → решение → результат (2–3 штуки)
- [ ] Отзывы клиентов
- [ ] FAQ: сроки, предоплата, правки
- [ ] Статус занятости («Свободен с …»)
- [ ] Локальное SEO: «разработчик Красноярск»
- [ ] Единая дизайн-система (цвета, типографика)

## P2 — рост

- [ ] Админка заявок (`contact_requests`, простой auth)
- [ ] Блог (`blog_posts` в существующей БД)
- [ ] Калькулятор стоимости
- [ ] Email-дайджест заявок (cron)
- [ ] Мультиязычность RU/EN
- [ ] Lighthouse 90+

## P3 — фриланс-биржи

- [ ] Страница «Для бирж» (Kwork, FL.ru)
- [ ] PDF-портфолио для скачивания
- [ ] Видео-презентация 1–2 мин
- [ ] Пакеты услуг: «Старт», «Бизнес», «Поддержка»

## Выполнено (фаза 1)

- [x] Контент под Красноярск и небольшие заказы
- [x] Страница контактов и форма
- [x] Backend API + PostgreSQL + Яндекс SMTP
- [x] Docker Compose + nginx + SSL-скрипты
- [x] BrowserRouter, документация, deploy.sh
- [x] Убраны учебные SPA-демо из меню; в портфолио: вёрстка, CyberZilla, RacketMate

## Чеклист: новые кейсы для портфолио

Каждый кейс оформить как: **задача → решение → результат** + скриншот + ссылка.

- [ ] **danilmakes** — сайт-визитка с формой, API, БД, деплой на VPS (оформить как кейс)
- [ ] **RacketMate** — описание на сайте (бот уже в проде)
- [ ] **CyberZilla** — краткое описание: что умеет, для чего подходит
- [ ] 1 новый pet-project «для витрины»: продукт с auth/БД или полезный сервис (не CRUD на чужом API)

Учебные SPA-демо (Posts list, Pokedex, Keksobooking) — **не показывать** заказчикам. Вёрстки — оставить в портфолио.
