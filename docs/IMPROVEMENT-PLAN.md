# План улучшений [danilmakes.ru](http://danilmakes.ru)

Стратегический план для выхода на фриланс: продуктовый разработчик без коммерческих кейсов под малый бизнес.

**Связанные документы:** [ROADMAP.md](./ROADMAP.md) · [SPEC-BACKLOG.md](./SPEC-BACKLOG.md)

**Прод:** [https://danilmakes.ru](https://danilmakes.ru)

---

## Контекст и ограничения


| Факт                                                           | Следствие для сайта                                                                |
| -------------------------------------------------------------- | ---------------------------------------------------------------------------------- |
| Продуктовый разработчик (Bnovo), фриланс — новый канал         | Позиционирование: мост «продукт → небольшие заказы», не «вебмастер с 10 клиентами» |
| Коммерческих фриланс-кейсов пока нет                           | Нужны витринные демо под типовые запросы + честная подача опыта                    |
| Вёрстка — учебные проекты (HTML Academy)                       | Свернуть/подписать явно, не выдавать за заказы                                     |
| RacketMate → VBallAgregator, проект остановлен на тестировании | Pet-project: показать продуктовый цикл, не активный прод                           |
| Bnovo-демо — сильный технический актив                         | Отдельная секция, не первая для SMB-аудитории                                      |
| Основная работа + подработка                                   | Уже в FAQ; не скрывать                                                             |


**Цель плана:** повысить конверсию «узнал себя» для малого бизнеса и дать второй вход для продуктовых/сложных задач — без выдуманных клиентов.

**Горизонт:** 6–8 недель (параллельно с откликами на биржах).

## Фаза 0 — Перепозиционирование (неделя 1)

Быстрые правки контента без новых проектов.

### 0.1 Hero и блок «Для кого»

**Сейчас:** «Разработчик в Красноярске для небольших проектов».

**Предложение:**

```
Продуктовый разработчик. Делаю сайты и небольшие приложения
для бизнеса — с понятным объёмом, рабочим UX и честными сроками.
```

**Файлы:** `src/pages/AboutMe/ui/AboutMe.tsx`, `index.html` (title, description).

**Критерий:** посетитель за 5 секунд понимает: (1) не студия, (2) есть продуктовый бэкграунд, (3) берёт небольшие заказы.

### 0.2 RacketMate — честный статус

**Сделать:**

- В `portfolio.ts`: результат → «Бот разрабатывался и работал в продакшене; проект сейчас на паузе».
- Убрать или смягчить CTA «открыть бота», если бот не отвечает.
- Добавить развёрнутый кейс `/portfolio/racketmate` (см. фазу 1).

**Файлы:** `src/shared/consts/portfolio.ts`, `src/shared/consts/case-studies.ts`.

### 0.3 Секция вёрстки

**Сделать:**

- Подпись секции: «Учебные макеты — демонстрация вёрстки, не коммерческие проекты».
- Секция **свёрнута по умолчанию** (`defaultOpen: false` в `PortfolioSection`).
- Опционально: оставить 2 лучших (Mishka + Jewellery), остальные убрать из `PORTFOLIO_LAYOUTS`.

**Файлы:** `src/pages/Portfolio/ui/Portfolio.tsx`, `src/shared/consts/portfolio.ts`.

### 0.4 Порядок портфолио

**Сделать:**

1. Секция **«Приложения»** — выше «Вёрстки».
2. Порядок карточек в `PORTFOLIO_PRODUCTS`:
  - danilmakes.ru
  - (новые витринные демо — после появления)
  - RacketMate
  - Bnovo-демо (report-revenue, tariff-prices, divisions) — сгруппировать
  - Family Meal Planning (pet-project, в конце)

**Файлы:** `src/shared/consts/portfolio.ts`, `src/pages/Portfolio/ui/Portfolio.tsx`.

### 0.5 Bnovo-демо — перевод для SMB

**Сделать:** подзаголовок секции внутри «Приложения» или note у группы:

```
Сложные интерфейсы (опыт в продуктовой разработке).
Демо на синтетических данных — тот же подход применим к расписанию,
заказам, каталогу и внутренним панелям.
```

**Критерий:** владелец малого бизнеса не думает, что вы «только для отелей».

## Фаза 2 — Витринные демо под фриланс (недели 2–5)

Целенаправленные проекты под типовые запросы. Подпись на всех: **«Портфолио-демо · учебный проект под типовую задачу»**.

### 2.1 Лендинг локального бизнеса (P0)

**Задача:** закрыть главный пробел — «сайт для такого же бизнеса, как мой».

**Сценарий (выбрать один):** салон красоты / стоматология / частный мастер / репетитор.

**Состав:**

- 1 страница, адаптив, hero + услуги + отзывы (вымышленные) + форма + контакты
- React + Vite (как основной сайт) или статика в `public/`
- Базовое SEO: title, description, og:image
- Кейс на `/portfolio/local-landing`

**Оценка:** 1–2 недели.

**Критерий:** заказчик узнаёт типовую задачу «лендинг от 15к»; демо открывается с телефона без поломок.

### 2.2 Интеграция «Форма → Telegram + почта» (P0)

**Задача:** услуга «Интеграции от 5 000 ₽» без абстрактного описания.

**Состав:**

- Минимальная форма на демо-странице
- Backend или serverless: заявка в Telegram-бот / почту (можно переиспользовать паттерн `server/` danilmakes)
- Кейс: задача → схема → результат

**Оценка:** 3–5 дней.

**Критерий:** в кейсе есть схема потока данных; на демо видно успешную отправку (mock или реальный канал).

### 2.3 Простая админка записей (P1, опционально)

**Задача:** наследник идеи RacketMate без привязки к теннису — «расписание / заявки / слоты».

**Состав:** список записей, календарь или таблица, смена статуса. MSW, без backend.

**Оценка:** 1–2 недели.

**Критерий:** SMB понимает сценарий без жаргона hotel PMS.

### 2.4 Family Meal Planning

**Не развивать** как коммерческий кейс. Подпись: «Pet-project, архитектурный эксперимент». Оставить в конце списка.

## Фаза 3 — Каналы привлечения (недели 3–8, параллельно)

Операционные задачи без обязательного кода.

### 3.1 Биржи


| Площадка | Услуги для старта                              | Ссылка на сайт               |
| -------- | ---------------------------------------------- | ---------------------------- |
| Kwork    | Лендинг, доработка React/Vue, форма → Telegram | danilmakes.ru + PDF (фаза 4) |
| FL.ru    | Доработка, сложный фронт, legacy API           | кейсы Bnovo + danilmakes     |


**Пакеты (черновик):**

- **Старт** — лендинг 5 блоков, форма, адаптив, от 15 000 ₽
- **Доработка** — баг / фича по ТЗ, от 2 000 ₽
- **Интеграция** — форма, Telegram, почта, от 5 000 ₽

### 3.2 Локальная сеть

- 3–5 личных контактов (знакомые с бизнесом) — не питч, а «беру 1–2 проекта, вот сайт»
- Чаты Красноярска / профильные TG — по желанию, без спама

### 3.3 SEO (из SPEC O1–O2)

- Яндекс.Вебмастер + sitemap
- Google Search Console
- Запросы: «разработчик сайта Красноярск», «сделать лендинг Красноярск»

### 3.4 Первый платный заказ

**Стратегия пилота:** скидка 20–30% в обмен на кейс + отзыв + право показать сайт (если не NDA).

После сдачи — полноценный кейс в `portfolio.ts` + `case-studies.ts` + при возможности отзыв на главной.
