## Цель следующего этапа

**Повысить конверсию «узнал себя»** за счёт:

1. Разнообразия ниш в SMB-демо (не только салон).
2. Упаковки существующих демо в цельные сценарии («экосистема для услуг»).
3. Первого реального заказа с отзывом.
4. Усиления слабых якорей на главной (MVP, доработка, админка).

**Горизонт:** 8–10 недель (параллельно с откликами на биржах).

**Главный принцип:** не добавлять enterprise-демо ради количества — приоритет у коммерческого кейса и второй SMB-ниши.

## Фаза 5 — Быстрые правки контента (неделя 1)

Без новых проектов. Закрыть разрывы между портфолио и главной.

### 5.1 Якорь «Админка записей» на главной

**Проблема:** демо есть, в `pricing.ts` ссылки нет — SMB не видит сценарий «что после заявки».

**Сделать:** добавить пример к услуге «Лендинг» или отдельную строку в описании интеграции:

```
Пример: панель заявок для салона → /portfolio/booking-admin
```

**Файлы:** `src/shared/consts/pricing.ts`, при необходимости `freelance-packages.ts`.

**Критерий:** с главной можно дойти до админки за 1 клик.

### 5.2 Переупаковка MVP на главной

**Проблема:** якорь ведёт на VBallAgregator (pet-project, пауза, ниша спорт) — слабо для владельца салона/клиники.

**Варианты (выбрать один):**

| Вариант | Якорь | Плюс |
| ------- | ----- | ---- |
| A | Оставить VBall, добавить подпись «pet-project, архитектура под заказ» | Минимум работы |
| B | Заменить на «экосистема Линия» (лендинг + форма + админка) | Лучше для SMB |
| C | Отложить до живого Telegram-бота (фаза 7) | Максимальная релевантность |

**Рекомендация:** вариант B сейчас, вариант C — когда появится бот.

**Файлы:** `src/shared/consts/pricing.ts`.

### 5.3 Якорь «Доработка» для SMB

**Проблема:** пример `tariff-prices` пугает hotel-жаргоном.

**Сделать:** в карточке «Доработка» добавить вторую ссылку или заменить label:

```
Пример для бизнеса: админка записей
Пример для сложного UI: таблица цен и ограничений
```

**Файлы:** `src/shared/consts/pricing.ts` (расширить `PricingTier.example` → массив или `examples[]`).

### 5.4 Кейс «Экосистема для услуг»

**Проблема:** Линия + форма + админка — три отдельные карточки, воронка неочевидна.

**Сделать:** на `/portfolio` или в кейсе `local-landing` добавить блок «Связанные демо»:

```
Лендинг → форма записи → уведомление в Telegram → панель заявок
```

**Файлы:** `src/shared/consts/case-studies.ts`, `src/pages/CaseStudy/ui/CaseStudy.tsx` (опционально: `relatedDemos`).

**Критерий:** посетитель понимает полный путь клиента без объяснений.

### 5.5 Расширить узнаваемость «Линии»

В кейсе `local-landing` явно указать: подходит для салона, барбершопа, косметолога, маникюрного кабинета.

**Файлы:** `src/shared/consts/case-studies.ts`, `portfolio.ts`.

## Фаза 6 — Первый коммерческий кейс (недели 1–4, параллельно)

**Приоритет P0.** Без него остальные улучшения дают потолок по доверию.

### 6.1 Пилотный заказ

**Стратегия:** скидка 20–30% в обмен на кейс + отзыв + право показать сайт (если не NDA).

**Каналы:**

- 3–5 личных контактов («беру 1–2 проекта, вот сайт»)
- Kwork / FL.ru — отклики с ссылкой на `/for-freelance`
- Локальные чаты Красноярска — без спама

**Подходящие первые заказы:**

| Тип | Почему |
| --- | ------ |
| Лендинг знакомого | Быстрый кейс, можно показать целиком |
| Доработка формы / Telegram | Малый объём, reuse `form-integration` |
| Фикс на Tilda / WordPress | Типовой запрос с бирж |

### 6.2 Оформление после сдачи

**Сделать:**

1. Карточка в `portfolio.ts` — **без** пометки «портфолио-демо».
2. Развёрнутый кейс в `case-studies.ts`: задача заказчика, сроки, результат.
3. Секция отзывов на главной (1 карточка).
4. Обновить PDF (`/portfolio-print`) и `/for-freelance`.

**Файлы:** `portfolio.ts`, `case-studies.ts`, `AboutMe.tsx`, новый `src/shared/consts/testimonials.ts`.

**Критерий:** на главной есть хотя бы один реальный отзыв с именем/нишей (с согласия).

## Фаза 7 — Второе SMB-демо в другой нише (недели 2–4)

**Приоритет P0.** Закрыть пробел «только салон красоты».

### 7.1 Выбор ниши

| Ниша | Спрос на биржах | Отличие от «Линии» |
| ---- | --------------- | ------------------ |
| **Стоматология / клиника** | Высокий | Доверие, врачи, акции, запись |
| **Частный репетитор** | Высокий | Расписание, предметы, цены за занятие |
| **Ремонт / строительство** | Средний | Калькулятор, портфолио работ |
| **Юрист / бухгалтер** | Средний | Сдержанный стиль, консультация |

**Рекомендация:** стоматология или репетитор — максимальный охват Kwork.

### 7.2 Состав демо

- 1 страница, адаптив, mobile-first
- Hero + услуги + отзывы (вымышленные) + форма + контакты
- Переиспользовать `POST /api/demo-lead` (как у «Линии»)
- Баннер: «Портфолио-демо · учебный проект под типовую задачу»
- Базовое SEO: title, description, og:image
- Кейс `/portfolio/<slug>`

**Оценка:** 1–2 недели.

**Файлы:** `public/<nicheLanding>/`, `app/codeExamples`, `portfolio.ts`, `case-studies.ts`, `pricing.ts` (второй пример или ротация).

**Критерий:** заказчик из другой ниши узнаёт «сайт для такого же бизнеса, как мой».

## Фаза 8 — Расширение охвата запросов (недели 4–8)

### 8.1 Интеграция amoCRM / Битрикс24 (P1)

**Задача:** второй по частоте запрос в РФ после «форма в Telegram».

**Состав:**

- Демо-страница: форма → API → CRM (mock или тестовый webhook)
- Схема потока (как у `form-integration`)
- Кейс `/portfolio/crm-integration`

**Оценка:** 3–7 дней.

**Файлы:** `public/crmIntegration/`, `server/` (новый endpoint или расширение `demo-lead`).

### 8.2 Многостраничный сайт (P1)

**Задача:** запрос «не лендинг, а сайт компании» (3–4 страницы).

**Состав:** Главная + Услуги + О компании + Контакты; общая шапка/подвал; одна ниша (B2B-услуги или производство).

**Оценка:** 2 недели.

### 8.3 Каталог услуг с фильтром (P2)

**Задача:** мастера и магазины с большим меню.

**Состав:** сетка карточек, фильтр по категории, кнопка «заявка» / «в корзину-lite».

**Оценка:** 1–2 недели. Можно переиспользовать паттерны Mishka/Jewellery без выдачи их за заказы.

### 8.4 Калькулятор / квиз (P2)

**Задача:** лид-магнит для услуг («расчёт стоимости ремонта»).

**Состав:** 3–5 шагов, итог + форма заявки → Telegram.

**Оценка:** 3–5 дней.

### 8.5 Telegram-бот для записи — живой мини-MVP (P2)

**Задача:** заменить слабый якорь VBallAgregator на работающий сценарий.

**Состав:** выбор услуги → дата/время → подтверждение; без мульти-спорта; деплой на VPS или serverless.

**Оценка:** 1–2 недели. Частично reuse VBallAgregator.
